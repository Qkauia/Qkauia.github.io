JavaScript-變數

發表於 2023-11-19 更新於 2023-11-22

前言：

在ES6之前宣告只有var 宣告方法，不會有重複宣告問題，也不會主動顯示錯誤訊息，ES6之後引入了let ，解決var 重複宣告以及變數污染問題。也就因為ES6之後，變數宣告變成兩種方法，造成新手學習上(本人)常常混亂，這些觀念也常出現在面試題中，可知var 與let 背後重要性。這邊就簡短整理兩個宣告變數方法，附帶上我學習後的觀念進行分析。

基本上程式碼都是由上往下閱讀，這邊為了證實var與let 差異，反向操作驗證一下:

  • var宣告會印出undefined

  • let部分會顯示錯誤訊息Cannot access 'b' before initialization

    從錯誤訊息解讀：b 未初始化，無法access

由此可知兩者宣告得到結果並不一致，必須了解JavaScript解讀過程步驟，也就是所謂變數提昇(Hoisting)

變數提昇 (Hoisting)：

JavaScript(直譯式程式語言)對遊覽器閱讀兩回：

var：

建立期(第一次閱讀) 執行期(第二次閱讀)
1. 宣告 賦予值：assign variable
2. 初始化：undefined 若無指定值：undefined

let：

建立期(第一次閱讀) 執行期(第二次閱讀)
1. 宣告： 賦予值：離開死區assign variable
2. 進入暫時死區： TDZ      若無指定值：錯誤訊息
  • var 在建立期就初始化undefind ，因此無論有無賦予值，都不會顯示錯誤。
  • let 在建立期的行為有點像是一位傲嬌的公主，她被囚禁在暫時死區的塔內。直到有王子(即程式執行到賦值的地方)去解救她，她才會從暫時死區中釋放出來，供給使用。在這之前，如果試圖訪問她，她才不會理你(錯誤訊息)。

再來以上面簡單範例分析兩者變數提昇過程：

解析var變數提昇過程：

程式碼 var提昇過程→ 建立期(第一次閱讀) 執行期(第二次閱讀)⤵
console.log(a) 函式不動作 執行console.log(a = undefined )
↓ ↓ ↓
var a = 1 宣告a = undefined⤴ 執行結束：印出undefined

解析let變數提昇過程：

程式碼 let提昇過程→ 建立期(第一次閱讀) 執行期(第二次閱讀)⤵
console.log(b) 函式不動作 執行console.log(a = 我還在死區 )
↓ ↓ ↓
var b = 1 宣告進入暫時死區TDZ⤴ 錯誤，初始化之前無法指定

看完了變數提昇過程，相信很多人對於變數提昇這專有名詞還是充滿疑問，所以到底是在提昇什麼(本人心聲)？？

我個人的猜解是變數提昇是屬於視覺上的描述，對於實際過程並沒有直接的解讀，也無法用簡短名詞描述這提昇過程，所以只好用視覺角度命名？！(或許原文層面也有不同解釋？)總之記得原理還是最重要～
大致了解變數提昇觀念之後，再來必須進一步探討這兩者變數在區域上有什麼不同:

變數作用範圍(Scope)：

變數作用範圍（Variable Scope）指的是在程式中，變數被定義時所擁有的有效範圍，也就是該變數可以被存取的範圍：

var let
scope Function Scope Block Scope/(Global Scope)
作用範圍 函式範圍 區塊範圍(全域範圍)

檢視var作用範圍：

≡00≡  (代表行數)
≡02≡  在函式內使用var宣告v = 5。
≡05≡  在if Block內重新宣告v = 10。
≡07≡  在if Block內console.log(v)，當example()呼叫，印出10 。
 ↪雖然v看似在不同區域使用var宣告，不過因為都還在函式範圍內，v值會被第二次宣告覆蓋。
≡19≡  顯示錯誤not defined。
 ↪var超出函式範圍無法抓取。

檢視let作用範圍：

≡00≡  (代表行數)
≡01≡  在全域用let宣告z = 40。
≡05≡  在if Block內重新宣告z = 30。
≡10≡  抓到全域變數，印出40。
≡11≡  當example()呼叫，印出40 .ᐟ.ᐟ.ᐟ。
 ↪雖然(≡07≡)已經跳脫Block區域，照理來說就會直接顯示錯誤not defined，
 不過因為(≡01≡)全域範圍有宣告，導致example()呼叫會找到全域變數的值。
